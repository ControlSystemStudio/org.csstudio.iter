ITER CSS Changelog

Version numbers in here refer to CODAC Core System.

Version 4.1 - 2013-06-15

  • New ITER CSS Getting Started section on the online Help
  • CSS Changelog – new ITER CSS section that lists the main newly integrated enhancements and bug fixes
  • New ITER CSS Tutorials – the exercises are also demonstrated during CSS training sessions
  • New PV Name lookup – auto completion based on History, DB files parsing and SDD information
  • New Electronic Logbook (ELog) – with automatic content from OPI, plot and alarm views
  • New XML Editor – to edit for instance OPI files directly in XML
  • BOY – Addition of a small square handle on top left of PV widget to allow editing PV Name directly on widget. Hover mouse on the handle will show PV Name as tooltip. Handle color could be grey or orange, depending on if the PV name is empty or not
  • BOY – Addition of a feedback to Grouping Container widget showing if it is locked. Click the handle can lock/unlock the container
  • BOY – Change of the looking and colours of Check Box and Radio Box to have modern style and feeling
  • BOY – Dump PV List context menu in Runtime to dump all PVs in the OPI including the ones for rules and scripts
  • Top OPIs menu allows now to open directly plot files
  • New Web Alarm Interface to consult the Alarm Tree and Alarm Table
  • Web Data Browser: the Search view is hidden by default. The vertical and horizontal zoom cursors have different shapes

Version 4.0 - 2013-02-15

  • Re-integration of CaSnooper - simple program to display statistics of EPICS broadcasts on the network.
  • Integration of PV Fields Viewer - diagnostic tool that displays information about a Process Variable. By default it displays data obtained by connecting to the life control system channel, which includes the name of the host that serves the channel, the underlying data type etc. ITER site-specific extension retrieves information from EPICS db files.
  • Integration of PACE Editor - this tool provides a tabular view of Process Variables (PV) names and their current values. The user can edit the values, change multiple values at once, review the changes and revert them if needed.
  • Integration of the Alarm Annunciator tool – the alarm description is converted to speech and annunciated.
  • Email Support Library integration - allows tools to send electronic mails with optional text and image attachments. For example, the Data Browser can email a screen-shot of the current plot.
  • Web OPI integration - aims to provide web access to Operator Interfaces (OPI) that were created in CSS BOY. With WebOPI, the user can seamlessly access to their OPIs from anywhere in the world at any time via a web-browser-equipped device such as a PC, laptop, tablet or smart phone.
  • Web Data Browser development – it is now possible to access to plotted data from a Web Navigator.
  • BOY: a PID example is provided at /opt/codac/opi/boy/resources/PID_Water_Cooling.opi.
